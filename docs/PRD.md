# 蓝心AI学习伴侣 MVP PRD

## 1. 项目定位

蓝心AI学习伴侣（Smart Study Companion）是面向备考、高强度学习、自习和远程学习场景的 AI 伴学 Demo。MVP 目标不是做完整商业产品，而是在可演示范围内跑通一条清晰闭环：

```text
开始学习 → 实时检测 → 分心 / 疲劳干预 → AI 伴聊 → 学习日报
```

项目核心价值是把传统“学习计时器”升级为“识别-干预-复盘”的主动式学习伴侣：

- 识别：通过端侧感知或 Mock 规则输出专注分、疲劳指数、分心等级。
- 干预：根据 L1 / L2 / L3 分级策略给出低打扰提醒。
- 复盘：在休息期通过 AI 伴聊引导用户口头或文字复盘，并生成学习日报。
- 隐私：MVP 先使用抽象状态数据与 Mock 模式，策划中真实人脸图像本地处理、即用即毁的原则作为后续落地说明。

## 2. 目标用户

| 用户类型 | 典型场景 | 核心诉求 |
|---|---|---|
| 考研 / 考公 / 考证学生 | 图书馆、自习室、宿舍长期备考 | 减少分心、获得阶段反馈、缓解学习焦虑 |
| 高校学生 | 课程复习、论文阅读、网课学习 | 判断自己是否真正专注，而不是只记录时长 |
| 知识工作者 | 深度阅读、技能学习、远程办公 | 保持专注节奏，避免疲劳硬撑 |
| 长期自学用户 | 居家自学、线上课程 | 获得低压力陪伴和持续激励 |

## 3. 核心痛点

1. 传统专注工具只记录学习时长，无法判断是否真的有效学习。
2. 普通提醒方式生硬，容易打断节奏或制造焦虑。
3. 学习结束后缺少复盘闭环，用户难以优化下次学习策略。
4. 高强度学习期间容易疲劳、孤独和低效硬撑，需要温和陪伴。
5. 摄像头、语音等敏感数据容易引发隐私担忧，因此 MVP 必须坚持 Mock 优先、抽象数据优先。

## 4. MVP 功能范围

### 4.1 必做功能

| 功能 | 说明 | 负责人 |
|---|---|---|
| 开始 / 结束学习 | 用户进入和退出学习会话 | Task 1 前端 |
| 学习计时 | 展示已学习秒数 / 分钟数 | Task 1 前端 |
| 实时状态展示 | 展示专注分、疲劳指数、分心次数、当前状态 | Task 1 前端 + Task 2 感知 |
| Mock StudyState | 每 5 秒输出一次标准学习状态 | Task 2 感知 |
| 专注分计算 | 0-100 分，专注分大于 70 计入有效学习 | Task 2 感知 |
| 疲劳指数计算 | 0-100 分，超过 60 触发疲劳提醒 | Task 2 感知 |
| L1 / L2 / L3 分级干预 | 轻提醒、中提醒、强提醒 | Task 1 前端 + Task 3 AI |
| AI 休息伴聊 | 疲劳或休息阶段生成温和对话和复盘引导 | Task 3 AI |
| 学习日报 | 展示时长、有效学习、专注曲线、AI 总结和建议 | Task 1 前端 + Task 3 AI |
| Mock 降级 | 没有真实摄像头和 vivo AIGC 接口时仍可完整演示 | 所有任务 |

### 4.2 非 MVP 功能

以下功能只作为答辩扩展点，不阻塞 Demo：

| 功能 | MVP 处理方式 |
|---|---|
| 真实 CameraX / MediaPipe 人脸识别 | 先用 Mock 和规则引擎替代 |
| BlueLM-V-3B 端侧多模态推理 | 作为技术路线说明，MVP 可用 Mock AI |
| vivo 原子组件 / 负一屏 | 可画静态入口或后续扩展，不影响主流程 |
| V-Touch 触觉反馈 | 可在文档说明，不强制实现 |
| 蓝心图像生成成就徽章 | 非必做，可用占位徽章 |
| 周报趋势 | 非必做，MVP 只做学习日报 |
| 完整账号体系和云同步 | 不做 |
| 复杂课程管理 / 题库 / 解题辅导 | 不做 |

## 5. 用户主流程

```text
1. 用户打开首页，看到今日学习时长、AI 鼓励语和“开始专注”按钮。
2. 用户点击“开始专注”，系统创建 sessionId，进入专注学习页。
3. 感知模块每 5 秒输出 StudyState。
4. 前端实时展示 elapsedSeconds、focusScore、fatigueScore、distractionCount。
5. 当 distractionLevel 为 L1 / L2 / L3 时，前端请求 AI 干预接口并展示对应提醒。
6. 当 fatigueScore 超过 60 或用户主动结束学习时，进入休息 / 伴聊页。
7. AI 生成休息伴聊开场白，并引导用户输入一句复盘。
8. 用户结束学习后，系统汇总状态序列和复盘内容，请求生成 StudyReport。
9. 前端展示学习日报，完成“识别-干预-复盘”闭环。
```

## 6. 页面清单

| 页面 | MVP 内容 | 接口依赖 |
|---|---|---|
| 首页 Dashboard | 今日学习时长、有效学习时长、AI 鼓励语、开始专注按钮 | `POST /api/study/start` |
| 专注学习页 | 计时器、专注分、疲劳指数、分心次数、当前状态、结束按钮 | `GET /api/study/state` |
| 分级干预弹窗 | L1 Toast、L2 中弹窗、L3 强确认弹窗 | `POST /api/ai/intervention` |
| 休息期 AI 伴聊页 | 休息倒计时、AI 关怀语、复盘输入框 | `POST /api/ai/rest-chat`、`POST /api/ai/oral-review` |
| 学习日报页 | 总时长、有效时长、平均专注分、疲劳峰值、曲线、总结建议 | `POST /api/ai/report`、`GET /api/study/report/{sessionId}` |
| 设置页 | 学习目标、提醒强度、隐私说明、Mock 模式开关 | 本地配置即可 |

## 7. 状态与干预规则

| 状态 | 条件 | 前端表现 |
|---|---|---|
| 深度专注 | `focusScore >= 80` 且 `fatigueScore < 60` | 蓝 / 绿状态，鼓励继续 |
| 稳定学习 | `60 <= focusScore < 80` | 蓝色状态，正常展示 |
| 轻微分心 | `distractionLevel = L1` | 顶部轻提示，不中断学习 |
| 连续分心 | `distractionLevel = L2` | 中等弹窗，可播放 AI 语音 |
| 严重分心 | `distractionLevel = L3` | 强弹窗，建议暂停和重新规划 |
| 疲劳偏高 | `fatigueScore >= 60` | 暖橙提醒，建议短休息 |

## 8. MVP 数据接口

接口以 Mock 优先为原则。前端只依赖项目标准接口，不直接依赖 vivo 官方 API，不直接读取感知模块内部实现。

核心数据结构：

- `StudyState`：感知模块输出的学习状态。
- `InterventionEvent`：AI 服务输出的干预事件。
- `StudyReport`：AI 服务输出的学习日报。

完整字段与接口路径见 [API.md](./API.md)。

## 10. 验收标准

### 10.1 产品验收

| 验收项 | 标准 |
|---|---|
| 主流程 | 必须完整跑通：开始学习 → 实时检测 → 分心 / 疲劳干预 → AI 伴聊 → 学习日报 |
| MVP 范围 | 不被真实摄像头、真实 vivo 接口、原子组件等增强功能阻塞 |
| 文案语气 | 温和、克制、不责备、不制造焦虑 |
| 视觉方向 | 极简、低饱和蓝白、圆角卡片、柔和渐变、陪伴感 |
| 隐私表达 | Demo 使用抽象状态和 Mock，强调敏感数据端侧处理 |

### 10.2 技术验收

| 模块 | 标准 |
|---|---|
| 前端 | 所有页面能使用 Mock 或接口数据完成演示 |
| 感知 | 每 5 秒输出符合 `StudyState` 的 JSON，支持至少 5 个 Mock 场景 |
| AI / 后端 | 返回符合 `InterventionEvent` 与 `StudyReport` 的标准 JSON，不暴露 vivo 原始响应 |
| 集成 | 前端、感知、AI 使用同一个 `sessionId` 串联 |
| Mock | 没有真实接口时仍可独立演示全流程 |

### 10.3 答辩验收

| 验收项 | 标准 |
|---|---|
| 创新点 | 能说明认知负荷调节、情感陪伴、端侧隐私、vivo 生态融合 |
| 大模型应用 | 能说明 AI 干预、伴聊、复盘、日报生成的场景 |
| 可行性 | 能说明 MVP 先 Mock 闭环，后续替换真实感知和 vivo AIGC |
| 分工 | 四个人清楚自己的输入、输出和验收标准 |
