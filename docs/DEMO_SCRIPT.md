# 蓝心AI学习伴侣 Demo 演示脚本

## 1. 演示目标

用 3-5 分钟展示 MVP 主流程：

```text
开始学习 → 实时检测 → 分心 / 疲劳干预 → AI 伴聊 → 学习日报
```

讲解重点：

1. 这不是普通计时器，而是“识别-干预-复盘”的 AI 学习伴侣。
2. MVP 采用 Mock 优先，因此没有真实摄像头和 vivo AIGC Key 时也能稳定演示。
3. 后续可把 Mock 感知替换为 CameraX / MediaPipe，把 Mock AI 替换为 vivo AIGC / 蓝心大模型接口。

## 2. 演示准备

| 准备项 | 要求 |
|---|---|
| Mock 模式 | 开启 |
| 学习任务 | 考研英语阅读 |
| 目标时长 | 45 分钟 |
| Mock 场景 | 正常专注、L1 轻微分心、L2 连续分心、疲劳偏高、L3 离座、恢复专注 |
| 日报数据 | 使用固定示例，保证演示稳定 |

## 3. 开场话术

```text
蓝心AI学习伴侣面向备考和高强度学习人群，目标是解决三个问题：
第一，只计时但不知道是否真的专注；
第二，提醒生硬，容易打断学习；
第三，学习结束后缺少复盘。

所以我们做了一个“识别-干预-复盘”的闭环：
开始学习后，系统实时检测专注分和疲劳指数；
当出现分心或疲劳时，AI 按 L1 到 L3 分级干预；
休息阶段 AI 主动伴聊并引导复盘；
最后自动生成学习日报。
```

## 4. 步骤一：首页展示

### 操作

打开 App 首页。

### 页面应展示

- 今日学习总时长。
- 今日有效学习时长。
- 当前 AI 鼓励语。
- “开始专注”按钮。

### 讲解

```text
这是首页，用户可以快速看到今天的学习状态和 AI 鼓励语。
MVP 里我们把入口做得很轻，只需要点击“开始专注”就能进入学习模式。
```

## 5. 步骤二：开始学习

### 操作

点击“开始专注”。

### 页面应展示

- 学习计时开始。
- 进入专注学习页。
- 初始状态为正常专注。

### Mock 数据

```json
{
  "focusScore": 88,
  "fatigueScore": 28,
  "distractionLevel": "NONE",
  "emotion": "focused"
}
```

### 讲解

```text
点击开始后，系统创建一次学习会话，并开始每 5 秒读取学习状态。
这里展示的专注分、疲劳指数和分心次数来自统一的 StudyState 数据结构。
```

## 6. 步骤三：实时状态监测

### 操作

停留在专注学习页，展示状态数据刷新。

### 页面应展示

- elapsedSeconds 增加。
- focusScore、fatigueScore 实时变化。
- 当前状态为“深度专注”或“稳定学习”。

### 讲解

```text
当专注分高于 70 时，这段时间会计入有效学习时长。
相比普通计时器，蓝心AI学习伴侣关注的是“有效学习”，而不是单纯坐了多久。
```

## 7. 步骤四：L1 轻微分心干预

### 操作

切换 Mock 场景为 `light_distraction`。

### Mock 数据

```json
{
  "focusScore": 66,
  "fatigueScore": 36,
  "distractionLevel": "L1",
  "gazeAwaySeconds": 25,
  "triggerReason": "视线偏离超过20秒"
}
```

### 页面应展示

- 顶部轻提示 / Toast。
- 不遮挡主要学习界面。

### 示例文案

```text
稍微有点分心了，先回到当前任务吧。
```

### 讲解

```text
当用户只是轻微分心时，系统使用 L1 轻提醒。
它不会强行打断学习，只是温和地把注意力拉回当前任务。
```

## 8. 步骤五：L2 连续分心干预

### 操作

切换 Mock 场景为 `heavy_distraction`。

### Mock 数据

```json
{
  "focusScore": 45,
  "fatigueScore": 48,
  "distractionLevel": "L2",
  "gazeAwaySeconds": 65,
  "triggerReason": "连续分心超过60秒"
}
```

### 页面应展示

- 中等弹窗。
- AI 语音按钮或语音状态。
- 一个具体行动建议。

### 示例文案

```text
你已经分心一小会儿了，先完成眼前这一小步，再休息一下。
```

### 讲解

```text
如果连续分心时间变长，系统会进入 L2。
这时 AI 会给出更明确的建议，但语气仍然温和，不责备用户。
```

## 9. 步骤六：L3 严重分心干预

### 操作

切换 Mock 场景为 `absent` 或 `entertainment_app`。

### Mock 数据

```json
{
  "focusScore": 30,
  "fatigueScore": 55,
  "distractionLevel": "L3",
  "isUserPresent": false,
  "triggerReason": "检测到离座或长时间无响应"
}
```

### 页面应展示

- 强弹窗。
- “暂停学习”或“重新开始当前任务”按钮。

### 示例文案

```text
当前状态不太适合继续硬撑，建议暂停一下，回来后重新开始这一小段。
```

### 讲解

```text
L3 用于离座、切换娱乐 App 或长时间无响应。
这一步体现的是行为问责闭环：不是惩罚用户，而是帮助用户重新规划学习节奏。
```

## 10. 步骤七：疲劳提醒与进入休息

### 操作

切换 Mock 场景为 `fatigue_high`，点击“进入休息”。

### Mock 数据

```json
{
  "focusScore": 72,
  "fatigueScore": 72,
  "distractionLevel": "NONE",
  "emotion": "tired",
  "triggerReason": "疲劳指数超过60"
}
```

### 页面应展示

- 暖橙色疲劳提示。
- 休息建议。
- 进入 AI 伴聊页。

### 讲解

```text
当疲劳指数升高时，系统不鼓励用户低效硬撑，而是建议短休息。
这对应策划书里的认知负荷调节：在用户可能超载时，主动降低学习压力。
```

## 11. 步骤八：AI 休息伴聊

### 操作

进入休息期 AI 伴聊页。

### 页面应展示

- AI 主动关怀语。
- 复盘引导问题。
- 用户输入框。

### 示例 AI 文案

```text
你已经坚持了45分钟，挺不容易的。现在可以先放松一下眼睛。
刚才这段英语阅读里，哪一类题最卡？
```

### 用户输入示例

```text
我完成了两篇英语阅读，但第二篇错得比较多。
```

### 讲解

```text
休息阶段不只是倒计时，AI 会用更像朋友的方式主动陪伴。
同时它会自然引导用户复盘，后续这句话会进入学习日报。
```

## 12. 步骤九：生成学习日报

### 操作

点击“结束学习 / 生成日报”。

### 页面应展示

- 总学习时长。
- 有效学习时长。
- 平均专注分。
- 最高疲劳指数。
- 分心次数。
- 专注曲线。
- AI 总结、优势、问题、建议、鼓励语。

### 示例日报数据

```json
{
  "totalMinutes": 45,
  "effectiveMinutes": 38,
  "averageFocusScore": 78,
  "maxFatigueScore": 66,
  "distractionCount": 4,
  "summary": "今天整体学习状态较好，有效学习时间占比较高。",
  "suggestions": [
    "下次可以把两篇阅读拆成两个25分钟任务",
    "第二篇阅读前安排3-5分钟休息，降低疲劳影响"
  ]
}
```

### 讲解

```text
学习结束后，系统把学习时长、专注曲线、分心次数、疲劳峰值和用户复盘整合成日报。
这样用户不仅知道自己学了多久，也知道哪里做得好、哪里可以优化。
```

## 13. 收尾话术

```text
这个 Demo 的重点是先跑通完整闭环。
当前版本使用 Mock 感知和 Mock AI 保证稳定演示；
后续可以把感知层替换为 CameraX + MediaPipe，把 AI 层替换为 vivo AIGC / 蓝心大模型接口。

所以蓝心AI学习伴侣的核心不是单点功能，而是：
端侧状态感知负责“看见学习状态”，
大模型负责“温和干预和总结”，
前端负责“低压力呈现”，
最终形成主动督学、情感陪伴、习惯养成和隐私保护的 AI 伴学闭环。
```

## 14. 演示验收清单

| 步骤 | 是否必须 | 通过标准 |
|---|---|---|
| 首页展示 | 是 | 能看到开始入口 |
| 开始学习 | 是 | 成功进入专注页并开始计时 |
| 实时状态监测 | 是 | 能展示 `StudyState` |
| L1 干预 | 是 | 轻提示出现，不打断 |
| L2 干预 | 是 | 中弹窗出现，有 AI 文案 |
| L3 干预 | 是 | 强弹窗出现，有确认动作 |
| 疲劳提醒 | 是 | 疲劳高时建议休息 |
| AI 伴聊 | 是 | 能展示伴聊和复盘问题 |
| 学习日报 | 是 | 能展示 `StudyReport` |
| 主流程 | 是 | 全程无崩溃、无接口断点 |
